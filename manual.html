画面を見ながら進められる、実務寄りの操作ガイド

DROP OFF の基本操作を、今の画面構成に合わせて整理しています。PDFではなくブラウザでそのまま見られます。 新規登録直後の流れから、予定表、実際の送り、自動配車結果、設定までを順番に確認できます。

最初に見るポイント

  • 新規登録直後は、ホームの「新規で作る」から開始します。
  • 起点を決めると、距離計算・地図・配車の基準が揃います。
  • 実際の送りは「予定から追加」と「直接入力」のどちらでも運用できます。

  • 個人情報にあたる箇所は一部ぼかしています。
  • 現在の運用に合わせて整理しています。

① 新規登録で最初に出るホーム画面

最初の一歩は、ホームの「新規で作る」から

新しいチームを作成した直後は、必要な基本データがまだ空です。ここで初期登録を進めます。

この画面でやること

  1. まず 起点を作る を押して、基準になる起点を登録します。
  2. 次に 車両を作る で配車に使う車両を登録します。
  3. 最後に 送り先を作る で送り先を登録します。
起点0件なら最優先 初期構築の入口 他画面へ自然につながる
新規登録直後は、ホーム中央の「新規で作る」カードが案内役になります。
② 通常のホーム画面

今日の状況と月間実績を一目で確認

日々の入口になる画面です。今日の件数と、車両ごとの月間距離・出勤日数・1日平均を確認できます。

見方のポイント

  • 今日の状況 では、人数・車両数・Plan・Actual・完了・キャンセル件数を確認できます。
  • 当日運用へ / 予定表へ / 実際の送りへ の3つが、日常操作の主な入口です。
  • 月間（車両別） では、各車両の月間距離・出勤日数・1日平均を確認できます。
  • 確定（月間へ反映） を使うと、自動配車結果の日次距離が月間集計へ累積されます。
  • free / Paid で一部機能制御があります。
基本データが入ると、ホームは日々の運用ダッシュボードに切り替わります。
③ 起点管理画面

距離計算と配車の土台になる「起点」を管理

使う起点を選ぶと、その後の距離計算・地図・配車の基準が切り替わります。保存できる起点数は free で1件、Paid で5件です。

操作の流れ

  1. 保存枠 を選びます。
  2. 起点名 と 住所 を入力します。
  3. APIで座標取得 または GoogleMapで開く を使って位置を確認します。
  4. この枠に保存 で保存します。
  5. 保存後は一覧の 使う を押して、現在の起点に切り替えます。
free は1件 Paid は5件 一覧から切替可能
住所から座標取得し、保存枠ごとに起点を管理します。
④ 車両管理画面

車両、ドライバー、帰宅方向、定員をまとめて管理

自動配車で使う車両情報を登録する画面です。配車後の月間距離や出勤日数の確認にも使います。

登録項目

  • 車両ID と ドライバー名 は基本情報です。
  • 担当方面 と 帰宅方面 は配車判断に使います。
  • 帰宅座標貼り付け を入れると、ラスト便や帰宅方向の判定に使えます。
  • 乗車可能人数 は同乗可能人数の上限です。
  • 下の一覧では、月間距離・出勤日数・1日平均も確認できます。

便利機能

  • インポート と 車両CSV は Paid で利用できます。
  • 下部の 走行実績エクスポート では、ドライバー別・日別の出力ができます。
車両登録と月間実績の確認が1画面にまとまっています。
⑤ 送り先管理画面

送り先候補を事前登録して、予定や実績に流し込む

名前、住所、座標、メモ、電話などを登録できます。距離と時間は、現在の起点を基準に都度再計算されます。新規の空フォームでは距離と時間は表示されません。

基本操作

  • 氏名 と 住所 を入力します。
  • 座標取得は API座標取得、Enter、手動座標貼り付けに対応しています。
  • 方面 は住所ベースの自動判定です。
  • ルート で地図確認、インポート と 送り先CSV は Paid で利用できます。
  • 送り先検索 タブから、既存データの絞り込みもできます。
  • free の API座標取得には日次上限があり、Paid は制限なしです。
送り先は事前登録しておくと、予定表や実際の送りで選びやすくなります。
⑥ 予定表

予定を先に組んでおき、時間帯ごとに一覧で管理

予定表は、当日の計画を先に作っておく画面です。あとから実際の送りへ引き継げます。

使い方

  1. 日付 を選びます。
  2. 送り先選択 で候補から選ぶか、必要に応じて直接入力します。
  3. 時間 を選び、必要なら 距離上書き を入れます。
  4. 方面 と メモ を確認して 追加/更新 します。

この画面でできること

  • 予定をエクスポート / 予定をインポート は Paid で利用できます。
  • 予定を全消去
  • 時間帯 × 方面一覧 で見やすく確認
予定表は、当日の配車前に全体像を組み立てるための下ごしらえ画面です。
⑦ 実際の送り

予定からでも、直接入力でも、実績側に登録できる

実運用で使う画面です。予定表から取り込む方法と、その場で直接登録する方法の両方に対応しています。

登録方法は2つあります

  • 予定から追加
    予定表で作成したデータを引き継いで実績化します。
  • 直接入力
    送り先選択からその場で登録します。予定がない突発対応でも使えます。

補足

  • 状態 は 未完了 / 完了 / キャンセル で管理します。
  • 表をコピー で一覧をコピーできます。
  • 下の 時間帯 × 方面一覧 では、色分けで状態が分かります。
予定を実績に変える入口にも、その場で直接登録する入口にもなります。
⑧ 自動配車結果

車両ごとの結果、地図、コピー、LINE送信をまとめて確認

自動配車を実行したあとに表示される結果画面です。車両別の配車順、累計距離、戻り時間、次便可能時刻などを確認できます。

見どころ

  • 車両稼働タイムライン で、戻り何分後・次便可能時刻・回転時間・累計距離/件数を確認できます。
  • 全配車 俯瞰マップ では、車両ごとの線やピンを切り替えられます。
  • 各車両カードでは、人数・累計距離・累計時間・累計件数 を確認できます。
  • 結果をコピー や LINE送信 で、ドライバー連携にも使えます。
  • 確定（月間へ反映） を押すと、当日の結果が日次実績へ保存され、同日2回目以降も累積されます。
配車の全体像と、各車両の詳細を1つの画面で確認できます。
⑨ 設定

ログイン情報、権限、招待、起点管理を確認

設定では、現在のログイン情報と権限確認、ユーザー招待、所属チーム情報の確認などを行えます。

主な項目

  • ログインユーザー と 現在の権限 を確認できます。
  • ユーザー管理 では一覧の更新、権限変更、状態変更、削除ができます。
  • ユーザー招待 では、招待先メールアドレス、表示名、招待権限、所属チームを指定します。
  • 現在のプラン は free / Paid で表示されます。
  • CSVや GoogleMap API を使う機能は、プランに応じて利用可否が変わります。
掲載画像では、ログイン情報など個人情報にあたる部分を一部ぼかしています。
設定ではユーザー管理と招待、起点の確認や更新を行います。
